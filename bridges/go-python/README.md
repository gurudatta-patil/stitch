# Stitch - Go → Python

Seamless cross-language IPC: a Go parent process communicates with a Python
child process over newline-delimited JSON-RPC on stdin/stdout.

---

## Overview

```
┌─────────────────────┐   stdin (JSON-RPC requests)   ┌──────────────────────┐
│   Go parent process │ ─────────────────────────────► │  Python child process│
│   (Bridge client)   │ ◄───────────────────────────── │  (sidecar)           │
└─────────────────────┘   stdout (JSON-RPC responses)  └──────────────────────┘
```

- All messages are **newline-delimited JSON**.
- Request: `{"id": "<uuid>", "method": "name", "params": {...}}`
- Success: `{"id": "...", "result": {...}}`
- Error: `{"id": "...", "error": {"message": "...", "traceback": "..."}}`
- Child emits `{"ready":true}` as its very first stdout line.
- Child has a stdin-EOF watchdog thread so it exits when the parent dies.
- Parent installs SIGINT/SIGTERM handler and uses SIGTERM → SIGKILL (2 s)
  for graceful shutdown.

## Quick Start

### 1. Copy the templates

```
bridges/go-python/
├── template.client.go   ← copy into your Go project
└── template.sidecar.py  ← copy alongside your Go binary
```

### 2. Edit the placeholders in `template.client.go`

```go
const (
    SIDECAR_SCRIPT = "sidecar.py"  // path to your Python sidecar
    PYTHON_BIN     = "python3"     // Python interpreter
)
```

### 3. Add your handlers to `template.sidecar.py`

```python
def _handle_greet(params: dict):
    return {"greeting": f"Hello, {params['name']}!"}

HANDLERS = {
    "echo":   _handle_echo,   # built-in smoke test
    "greet":  _handle_greet,  # your handler
}
```

### 4. Call from Go

```go
bridge, err := NewBridge()
if err != nil { log.Fatal(err) }
defer bridge.Close()

result, err := bridge.Call("greet", map[string]interface{}{"name": "World"})
```

### 5. Go module dependency

```
go get github.com/google/uuid
```

## Protocol Details

### Ready handshake

The Go parent blocks in `waitReady()` reading the first line from the child.
It expects `{"ready":true}`.  Any other content (or EOF) is treated as a
startup failure and the child is killed.

### Request IDs

Every request carries a UUID v4 generated by `github.com/google/uuid`.
Responses are matched back to callers via a `map[string]chan rpcResponse`
protected by a `sync.Mutex`.  Multiple goroutines can call `Call` concurrently
without coordination on the caller side.

### Error propagation

Python exceptions are caught in the sidecar's `_dispatch` function, serialised
with `traceback.format_exc()`, and returned as:

```json
{"id": "...", "error": {"message": "ValueError: ...", "traceback": "..."}}
```

The Go client returns this as a Go `error` value.  The full traceback is
included in `error.Error()` so it appears in Go logs.

### Shutdown sequence

1. `bridge.Close()` closes the stdin pipe - Python's watchdog detects EOF and
   calls `os._exit(0)`.
2. The `done` channel is closed to unblock the `readLoop` goroutine.
3. `killChild()` calls `SIGTERM`, waits up to 2 s, then `SIGKILL`.
4. `cmd.Wait()` reaps the child and releases OS resources.
